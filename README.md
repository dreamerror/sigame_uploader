# SiGame Media Cutter

SiGame Media Cutter - локальная desktop-утилита для быстрой подготовки коротких аудио- и видеофрагментов для пакетов SiGame.

Основной сценарий: вставить ссылку YouTube, получить информацию о ролике, выбрать начало и конец фрагмента, затем экспортировать MP3 или MP4 в выбранную папку. Thumbnail можно скачать отдельно.

Приложение не является публичным сервисом скачивания и не дает прав на использование чужого контента. Используйте только медиа, на которое у вас есть права или разрешение.

## Возможности

- Ввод YouTube URL.
- Получение названия, длительности и thumbnail через `yt-dlp`.
- Preview-плеер с локальным streaming proxy.
- Timeline для выбора start/end без waveform.
- Точный ввод таймкодов до миллисекунд.
- Проигрывание выбранного отрезка.
- Экспорт MP3 и MP4 через `yt-dlp` и `ffmpeg`.
- Выбор качества MP4: 360p, 480p, 720p, 1080p или best.
- Выбор папки сохранения через системный dialog.
- Скачивание thumbnail в выбранную папку.
- Создание минимального `.siq` пакета из последнего экспортированного MP3/MP4.
- Отдельная вкладка базового редактора `.siq`: раунды, темы, вопросы, финальный раунд, открытие и сохранение пакета.
- Добавление медиа в вопрос из последнего экспорта или готового файла с диска.
- Preview медиа прямо в вопросе пакета через локальный proxy.
- Проверка доступности `yt-dlp`, `ffmpeg` и `ffprobe`.
- Русскоязычные статусы, сообщения об ошибках и подсказки.
- Опциональное использование cookies браузера для случаев, когда YouTube требует вход или anti-bot подтверждение.
- Опциональный локальный кэш cookies, которым пользователь управляет явно.

## Ограничения

- Сейчас поддерживается только YouTube.
- Для живого экспорта нужна сеть и доступная ссылка YouTube.
- `yt-dlp`, `ffmpeg` и `ffprobe` не встроены в приложение и должны быть установлены отдельно.
- Доступность YouTube зависит от сети, VPN, региона, настроек аккаунта и текущей работоспособности `yt-dlp`.
- Preview может быть менее стабильным, чем export: ссылка на поток может истечь, быть недоступной или не воспроизводиться media engine внутри Electron.
- Редактор `.siq` пока базовый: обычные вопросы, текст, ответ, цена, одно вложенное медиа в вопросе, сворачиваемое дерево структуры и сворачиваемые секции редактора.
- Нет спецтипов вопросов SiGame, одновременного запуска картинки и медиа, полноценного переноса всех возможностей SiQuester, waveform, очереди заданий, нарезки локальных файлов, Freesound/Jamendo и Яндекс.Музыки.

## Установка внешних инструментов

Приложение использует:

- `yt-dlp` - получает metadata и медиа-потоки YouTube;
- `ffmpeg` - обрабатывает и экспортирует фрагменты;
- `ffprobe` - анализирует медиа.

После установки проверьте инструменты в новом окне PowerShell:

```powershell
yt-dlp --version
ffmpeg -version
ffprobe -version
```

### Через winget

```powershell
winget install yt-dlp.yt-dlp
winget install Gyan.FFmpeg
```

### Через Chocolatey

```powershell
choco install yt-dlp
choco install ffmpeg
```

### Через Scoop

```powershell
scoop install yt-dlp
scoop install ffmpeg
```

### Ручная установка

1. Скачайте `yt-dlp.exe` и положите его, например, в `C:\Tools\yt-dlp\yt-dlp.exe`.
2. Скачайте Windows-сборку FFmpeg и распакуйте ее, например, в `C:\Tools\ffmpeg`.
3. Убедитесь, что в папке `C:\Tools\ffmpeg\bin` есть:
   - `ffmpeg.exe`;
   - `ffprobe.exe`.
4. Добавьте папки с `.exe` в системный `PATH` или укажите переменные окружения, как описано ниже.

## Как приложение ищет инструменты

Порядок поиска `yt-dlp`, `ffmpeg` и `ffprobe`:

1. Переменные окружения:
   - `YT_DLP_PATH`;
   - `FFMPEG_PATH`;
   - `FFPROBE_PATH`.
2. Стандартная папка ссылок winget:
   - `%LOCALAPPDATA%\Microsoft\WinGet\Links`.
3. Системный `PATH`:
   - `yt-dlp`;
   - `ffmpeg`;
   - `ffprobe`.

Пример временной настройки для текущего окна PowerShell:

```powershell
$env:YT_DLP_PATH="C:\Tools\yt-dlp\yt-dlp.exe"
$env:FFMPEG_PATH="C:\Tools\ffmpeg\bin\ffmpeg.exe"
$env:FFPROBE_PATH="C:\Tools\ffmpeg\bin\ffprobe.exe"

npm run dev
```

Такие переменные действуют только в этом окне PowerShell. Для постоянной настройки добавьте инструменты в системный `PATH` или создайте пользовательские переменные окружения Windows.

## Запуск приложения

### Dev-режим

```powershell
npm install
npm run check:tools
npm run dev
```

- `npm install` устанавливает зависимости проекта.
- `npm run check:tools` проверяет доступность `yt-dlp`, `ffmpeg` и `ffprobe`.
- `npm run dev` запускает Electron-приложение.

В dev-режиме интерфейс приложения загружается с локального dev-server:

```text
127.0.0.1:5173
```

Порт фиксированный. Если `5173` уже занят, dev-запуск завершится ошибкой.

### VPN и доступ к 127.0.0.1

При включенном VPN, proxy, TUN-режиме или приложениях для маршрутизации трафика локальные адреса должны быть доступны напрямую:

- `127.0.0.1`;
- `localhost`;
- желательно весь диапазон `127.0.0.0/8`.

Если VPN перехватывает или блокирует локальные адреса, окно Electron может не открыться, остаться пустым или не загрузить интерфейс.

Что проверить:

- добавьте `127.0.0.1/32` или `127.0.0.0/8` в direct/excluded routes VPN-клиента;
- убедитесь, что во время dev-запуска открывается `http://127.0.0.1:5173`;
- если используется Happ или похожий VPN-клиент, настройте локальные адреса напрямую, не через туннель;
- для проверки временно отключите VPN.

### Собранная версия

Собранное приложение запускается как обычное desktop-приложение, но внешние инструменты все равно должны быть доступны через `PATH`, winget Links или переменные окружения.

После `npm run package` unpacked-версия находится здесь:

```text
release/win-unpacked/SiGame Media Cutter.exe
```

После `npm run dist` portable `.exe` находится в папке:

```text
release
```

## Быстрый smoke-test

Для проверки нужен интернет, доступный YouTube, валидная ссылка и медиа, которое вы имеете право использовать.

1. Установите `yt-dlp`, `ffmpeg` и `ffprobe`.
2. Проверьте инструменты:

   ```powershell
   npm run check:tools
   ```

3. Запустите приложение:

   ```powershell
   npm run dev
   ```

4. Вставьте YouTube URL.
5. Нажмите кнопку получения metadata.
6. Убедитесь, что появились название, длительность и thumbnail.
7. Выберите start/end через поля времени или timeline.
8. Выберите папку сохранения.
9. Выберите формат MP3 или MP4.
10. Запустите export.
11. Проверьте, что итоговый файл появился в выбранной папке.

## Форматы времени

Поддерживаются таймкоды:

```text
SS
SS.mmm
MM:SS
MM:SS.mmm
HH:MM:SS
HH:MM:SS.mmm
```

Примеры:

```text
12
12.500
01:12
01:12.250
00:01:12.250
```

Правила:

- `start` не может быть меньше `0`;
- `end` должен быть больше `start`;
- минимальная длительность фрагмента - `0.5` секунды;
- если длительность ролика известна, `end` не должен быть больше duration.

## Базовый редактор `.siq`

Во вкладке "Пакет" можно создать новый пакет или открыть существующий `.siq` с базовой структурой:

1. Нажмите "Пакет" в верхней панели.
2. Создайте новый пакет или откройте существующий `.siq`.
3. Добавьте раунды, темы и вопросы. Для финального раунда используйте кнопку "+ Финал" или тип "Финальный".
4. Отредактируйте название пакета, автора, сложность, текст вопроса, цену и ответ.
5. Для медиа в вопросе можно использовать последний экспортированный MP3/MP4, выбрать готовый файл с диска или перейти во вкладку "Медиа", подготовить фрагмент и вернуться к пакету.
6. Выберите папку сохранения и сохраните `.siq`.

Состояние редактора пакета сохраняется при переключении между вкладками "Пакет" и "Медиа", поэтому можно спокойно подготовить фрагмент и вернуться к тому же черновику.

Выбранное медиа показывается в preview внутри вопроса, если это audio/video/image. При сохранении `.siq` медиафайл встраивается в пакет, поэтому после проверки пакета в SiQuester исходный файл можно удалить с диска.

Импорт существующих пакетов поддерживает базовую структуру SiQuester: раунды, темы, обычные вопросы, текст, ответ, комментарии и первое вложенное audio/video/image/html медиа. Сложные типы вопросов пока могут быть импортированы частично.

## Cookies браузера для YouTube

Иногда YouTube отвечает ошибкой вроде `Sign in to confirm you're not a bot`. В этом случае обычный запрос `yt-dlp` может не получить metadata, preview или поток для export.

В блоке "Доступ YouTube" можно:

1. Открыть вход YouTube и войти в обычном браузере.
2. Включить использование cookies из браузера для `yt-dlp`.
3. Выбрать браузер, где выполнен вход.
4. Повторить получение metadata или export.

Поддерживаются Chrome, Edge, Firefox, Brave, Яндекс.Браузер, Vivaldi, Opera и Chromium. Для Яндекс.Браузера приложение использует Chromium-профиль.

Если не хочется каждый раз обращаться к базе cookies браузера, можно включить локальный кэш cookies и обновить его отдельной кнопкой. Кэш хранит чувствительные сессионные данные локально в данных приложения, поэтому очищайте его, когда доступ больше не нужен.

## Troubleshooting

### `yt-dlp` не найден

Проверьте:

```powershell
yt-dlp --version
npm run check:tools
```

Если команда не найдена:

- откройте новое окно PowerShell после установки;
- проверьте, что `yt-dlp.exe` доступен через `PATH`;
- задайте `YT_DLP_PATH`, если используете ручную установку;
- при установке через winget проверьте `%LOCALAPPDATA%\Microsoft\WinGet\Links`.

### `ffmpeg` или `ffprobe` не найдены

Проверьте:

```powershell
ffmpeg -version
ffprobe -version
npm run check:tools
```

Если проверка не проходит:

- убедитесь, что `ffmpeg.exe` и `ffprobe.exe` есть в папке `bin` вашей FFmpeg-сборки;
- добавьте эту папку в `PATH`;
- или задайте `FFMPEG_PATH` и `FFPROBE_PATH`.

### Приложение не открывается в dev-режиме при включенном VPN

Dev-server использует `127.0.0.1:5173`. Локальные адреса должны идти напрямую, не через VPN-туннель.

Что сделать:

- исключите `127.0.0.1/32` или `127.0.0.0/8` из VPN-маршрутизации;
- проверьте `http://127.0.0.1:5173` во время dev-запуска;
- освободите порт `5173`, если он занят;
- временно отключите VPN для проверки.

### Metadata долго загружается или падает по timeout

Возможные причины:

- YouTube недоступен из текущей сети;
- VPN или proxy мешает запросам;
- `yt-dlp` устарел;
- конкретный ролик удален, приватный, ограничен по региону или требует вход.

Что попробовать:

```powershell
npm run debug:metadata -- "URL"
```

Также проверьте ссылку в браузере, обновите `yt-dlp` и попробуйте другой разрешенный тестовый ролик.

### YouTube просит подтвердить, что вы не бот

Если ошибка содержит `Sign in to confirm you're not a bot`, включите cookies браузера в блоке "Доступ YouTube":

- войдите в YouTube в выбранном браузере;
- включите использование cookies из браузера;
- выберите этот браузер в приложении;
- повторите metadata, preview или export.

Если чтение browser cookies нестабильно, обновите локальный кэш cookies и используйте его для следующих запросов.

### Export работает в dev-режиме, но не работает в собранном приложении

Частая причина: dev-режим запускался из PowerShell с временными переменными окружения, а `.exe` запущен без них.

Что сделать:

- добавьте `yt-dlp`, `ffmpeg` и `ffprobe` в системный `PATH`;
- или настройте постоянные переменные `YT_DLP_PATH`, `FFMPEG_PATH`, `FFPROBE_PATH`;
- перезапустите приложение после изменения переменных.

### Preview не работает, но export работает

Это возможно. Preview зависит от локального streaming proxy и media engine внутри Electron. Export использует `ffmpeg` и может успешно работать даже тогда, когда preview не воспроизводится.

Что попробовать:

- заново получить metadata;
- повторно подготовить preview;
- проверить сеть и VPN;
- выбрать start/end вручную и выполнить export.
