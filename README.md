# SiGame Media Cutter

Локальная некоммерческая настольная утилита для подготовки коротких аудиофрагментов для пакетов SiGame из медиа, которое пользователь имеет право использовать.

Приложение не является публичным сервисом скачивания и не должно позиционироваться как инструмент обхода авторских прав.

## MVP

- Пользователь вставляет ссылку YouTube.
- Приложение получает название, длительность и превью через `yt-dlp`.
- Пользователь вручную вводит начальную и конечную отметки времени.
- Приложение экспортирует выбранный фрагмент в MP3 или MP4 через `ffmpeg`.
- Thumbnail можно скачать отдельной кнопкой в выбранную локальную папку.
- Результат сохраняется в выбранную локальную папку.
- UI показывает понятное состояние успеха или ошибки.

Renderer не вызывает `child_process`. Все вызовы `yt-dlp`, `ffmpeg` и `ffprobe` находятся в сервисах main-процесса и доступны renderer только через типизированный preload API.

## Требования для Windows

Установите:

- Node.js 20 или новее
- `yt-dlp`
- `ffmpeg` вместе с `ffprobe`

Приложение ищет внешние инструменты так:

1. Сначала по переменным окружения:
   - `YT_DLP_PATH`
   - `FFMPEG_PATH`
   - `FFPROBE_PATH`
2. Затем в стандартной папке ссылок winget:
   - `%LOCALAPPDATA%\Microsoft\WinGet\Links`
3. Если инструмент не найден там, используется команда из системного `PATH`:
   - `yt-dlp`
   - `ffmpeg`
   - `ffprobe`

## Установка yt-dlp, FFmpeg и ffprobe

### Проверка после установки

Откройте PowerShell и выполните:

```powershell
yt-dlp --version
ffmpeg -version
ffprobe -version
```

Если все три команды выводят версии, приложение сможет найти инструменты через `PATH`.

Внутри проекта можно дополнительно выполнить:

```powershell
npm run check:tools
```

Эта команда учитывает `YT_DLP_PATH`, `FFMPEG_PATH`, `FFPROBE_PATH`, стандартную папку `%LOCALAPPDATA%\Microsoft\WinGet\Links`, а затем fallback на системный `PATH`.

### Установка через winget

Если у вас доступен `winget`, попробуйте:

```powershell
winget install yt-dlp.yt-dlp
winget install Gyan.FFmpeg
```

После установки откройте новое окно PowerShell и проверьте команды из раздела выше.

### Установка через Chocolatey

Если вы пользуетесь Chocolatey:

```powershell
choco install yt-dlp
choco install ffmpeg
```

После установки откройте новое окно PowerShell и проверьте:

```powershell
yt-dlp --version
ffmpeg -version
ffprobe -version
```

### Установка через Scoop

Если вы пользуетесь Scoop:

```powershell
scoop install yt-dlp
scoop install ffmpeg
```

После установки проверьте доступность команд в PowerShell.

### Ручная установка

1. Скачайте `yt-dlp.exe` и положите его, например, в `C:\Tools\yt-dlp\yt-dlp.exe`.
2. Скачайте сборку FFmpeg для Windows и распакуйте её, например, в `C:\Tools\ffmpeg`.
3. Убедитесь, что в папке `C:\Tools\ffmpeg\bin` есть оба файла:
   - `ffmpeg.exe`
   - `ffprobe.exe`
4. Добавьте папки с `.exe` в системный `PATH` или задайте переменные окружения перед запуском приложения.

Пример запуска с явными путями в PowerShell:

```powershell
$env:YT_DLP_PATH="C:\Tools\yt-dlp\yt-dlp.exe"
$env:FFMPEG_PATH="C:\Tools\ffmpeg\bin\ffmpeg.exe"
$env:FFPROBE_PATH="C:\Tools\ffmpeg\bin\ffprobe.exe"
npm run dev
```

## Локальный запуск

```powershell
npm install
npm run check:tools
npm run dev
```

Если `npm run check:tools` сообщает, что инструмент не найден, живой экспорт работать не будет до установки инструмента или настройки переменной окружения.

В dev-режиме renderer dev server принудительно запускается на IPv4:

```text
127.0.0.1:5173
```

Это сделано для более предсказуемого локального запуска при включённом VPN, когда `localhost` или IPv6 могут работать некорректно. Порт фиксированный; если `5173` занят, dev-запуск завершится ошибкой. Освободите порт или остановите другой Vite/dev-сервер.

## Smoke-test живого экспорта

Для проверки нужен рабочий интернет, валидная ссылка YouTube и медиа, которое пользователь имеет право использовать.

1. Запустите приложение:

   ```powershell
   npm run dev
   ```

2. Вставьте ссылку YouTube в поле “Ссылка YouTube”.
3. Нажмите “Получить”.
4. Во время запроса приложение покажет индикатор и таймер ожидания. Если сеть, VPN или YouTube тормозят, `yt-dlp` может отвечать не сразу.
5. Убедитесь, что появились название, длительность и превью.
6. Введите начало и конец фрагмента, например:
   - начало: `0:15`
   - конец: `0:45`
7. Нажмите “Выбрать” и выберите папку сохранения.
8. Проверьте строку “MP3/MP4 будет сохранён”: там показан ожидаемый путь итогового файла.
9. Выберите формат экспорта: MP3 или MP4.
10. Блок качества MP4 виден всегда, но активен только при выбранном MP4. Можно выбрать 360p, 480p, 720p, 1080p или “Лучшее”; по умолчанию используется 720p.
11. Нажмите “Экспорт MP3” или “Экспорт MP4”.
12. После успешного экспорта приложение покажет точный путь файла в статусе.
13. При необходимости нажмите “Скачать thumbnail”, чтобы сохранить обложку ролика в выбранную папку.

Если `yt-dlp`, `ffmpeg` или `ffprobe` не найдены, приложение покажет ошибку на русском с подсказкой: какую команду проверить в PowerShell и какую переменную окружения можно задать.

Если получение метаданных или аудиопотока занимает слишком много времени, запрос к `yt-dlp` останавливается по таймауту и приложение показывает ошибку вместо бесконечной загрузки. Сейчас приложение даёт `yt-dlp` до 120 секунд и принудительно использует IPv4 для этих запросов.

## Плеер и точный выбор фрагмента

После получения metadata приложение автоматически готовит preview. Также можно нажать “Подготовить preview” или “Обновить preview” вручную. Main-процесс получает media URL и нужные HTTP-заголовки через `yt-dlp`, затем отдаёт renderer локальную ссылку на streaming proxy `127.0.0.1`. Временные preview-файлы при этом не создаются. Если preview подготовить не удалось, ручной ввод start/end и экспорт MP3/MP4 остаются доступны.

Поддерживаемые форматы времени:

- `SS`
- `SS.mmm`
- `MM:SS`
- `MM:SS.mmm`
- `HH:MM:SS`
- `HH:MM:SS.mmm`

Примеры: `12`, `12.500`, `01:12`, `01:12.250`, `00:01:12.250`.

Внутри приложения время хранится в секундах с точностью до 3 знаков после точки. Для отображения используется формат `00:00.000`, `01:02.345` или `01:02:03.456`, если есть часы.

Timeline-шкала работает без waveform и показывает всю длительность, текущую позицию плеера и выбранный отрезок. Можно кликнуть по шкале для перемотки, перетаскивать start/end handles, менять поля вручную и использовать кнопки `-1с`, `-0.1с`, `+0.1с`, `+1с`.

Кнопки рядом с плеером позволяют поставить начало или конец по текущей позиции и сразу проиграть выбранный отрезок. Такие же операции доступны ниже, рядом с полями start/end. Кнопка “Проиграть отрезок” перематывает плеер на start, запускает playback и ставит его на паузу при достижении end. Это практическая точность браузерного media engine, не sample-accurate остановка.

Экспорт использует точные start/end значения из полей, включая миллисекунды. MP3 экспортирует аудиофрагмент, MP4 экспортирует видеофрагмент с аудио, если подходящий поток доступен через `yt-dlp`.

Для MP4 можно выбрать качество: 360p, 480p, 720p, 1080p или “Лучшее”. Блок качества виден рядом с форматом экспорта; при MP3 он отключён, чтобы настройка не выглядела пропавшей. Значение по умолчанию — 720p. Если выбранного качества нет, `yt-dlp` попытается подобрать ближайший подходящий progressive video+audio поток. Режим “Лучшее” может создать более крупный файл и работать дольше.

Thumbnail скачивается отдельной кнопкой в выбранную папку сохранения. Файл получает имя на основе названия медиа и расширение по ответу сервера или URL картинки.

## Запуск и тестирование desktop-сборки

### Dev-режим

```powershell
npm run check:tools
npm run dev
```

Dev-режим открывает настоящее окно Electron, а renderer загружается с локального dev-сервера `127.0.0.1:5173`. Это не публичный веб-сервис и не внешний адрес.

### Сборка без установщика

```powershell
npm run package
```

Команда сначала выполняет `npm run build`, затем собирает unpacked desktop-приложение через `electron-builder --dir`.

Сборка использует локальный Electron из `node_modules/electron/dist`, поэтому для unpacked-сборки не нужно заново скачивать Electron с GitHub после `npm install`.

Для локального MVP отключены подпись и редактирование Windows executable (`signAndEditExecutable: false`). Это помогает собирать тестовую desktop-версию без сертификатов, иконки и дополнительных прав на symlink в кэше `winCodeSign`.

Результат ищите в:

```text
release/win-unpacked/
```

Запуск:

```powershell
.\release\win-unpacked\SiGame Media Cutter.exe
```

### Portable-сборка

```powershell
npm run dist
```

Команда собирает portable `.exe` для Windows. Результат появляется в папке:

```text
release/
```

Имя файла формируется примерно так:

```text
SiGame Media Cutter-0.1.0-x64.exe
```

### Внешние инструменты в собранной версии

Собранное приложение пока не включает `yt-dlp`, `ffmpeg` и `ffprobe` внутрь себя. Для живого экспорта в dev-режиме и в собранной версии всё ещё нужны:

- установленный `yt-dlp`;
- установленный `ffmpeg`;
- доступный `ffprobe`;
- рабочая сеть;
- валидная ссылка YouTube;
- право пользователя использовать выбранный контент.

Поиск инструментов такой же, как в dev-режиме: `YT_DLP_PATH`/`FFMPEG_PATH`/`FFPROBE_PATH`, затем `%LOCALAPPDATA%\Microsoft\WinGet\Links`, затем системный `PATH`.

## Troubleshooting

### Dev-окно не открывается при включённом VPN

- Убедитесь, что запускаете актуальную версию проекта.
- Проверьте, что dev-server стартует на `127.0.0.1:5173`.
- Полностью перезапустите терминал или IDE после установки системных инструментов.
- Если VPN перехватывает localhost-трафик, временно отключите VPN для проверки.

### Порт 5173 занят

Dev-сервер использует `strictPort`, поэтому не переключается на случайный порт. Закройте другой dev-сервер/Vite-процесс, который занимает `5173`, и повторите:

```powershell
npm run dev
```

### `yt-dlp`, `ffmpeg` или `ffprobe` не найдены

Запустите:

```powershell
npm run check:tools
```

Если проверка не проходит:

- откройте новое окно PowerShell после установки через `winget`;
- проверьте, что `%LOCALAPPDATA%\Microsoft\WinGet\Links` содержит нужные `.exe`;
- задайте `YT_DLP_PATH`, `FFMPEG_PATH`, `FFPROBE_PATH` явно;
- проверьте команды `yt-dlp --version`, `ffmpeg -version`, `ffprobe -version`.

### После вставки ссылки долго висит “Загрузка”

- Посмотрите на индикатор и таймер: приложение ожидает ответ `yt-dlp`.
- Проверьте VPN и сеть: YouTube может быть недоступен или отвечать медленно.
- Запустите `npm run check:tools`, чтобы убедиться, что используется рабочий `yt-dlp`.
- Попробуйте ту же ссылку в PowerShell:

  ```powershell
  yt-dlp --dump-single-json --no-playlist --skip-download --socket-timeout 20 "ССЫЛКА_YOUTUBE"
  ```

Если команда в PowerShell тоже долго не отвечает или падает, проблема вне UI приложения.

Можно запустить проектный debug-скрипт, который использует те же правила поиска `yt-dlp`, что и приложение:

```powershell
npm run debug:metadata -- "ССЫЛКА_YOUTUBE"
```

Он покажет путь к `yt-dlp`, время выполнения, stderr/stdout при ошибке и базовые метаданные при успехе.

### Preview не запускается

- Проверьте, что metadata уже загружены.
- Нажмите “Подготовить preview” ещё раз: upstream-ссылка от YouTube может истечь.
- Проверьте VPN и сеть.
- Запустите `npm run debug:metadata -- "ССЫЛКА_YOUTUBE"`, чтобы проверить доступность ссылки через `yt-dlp`.
- Если плеер пустой или показывает ошибку, возможно, YouTube отдал поток или заголовки, которые Electron media engine всё ещё не может воспроизвести даже через локальный proxy. Export может при этом работать.

### MP4 export не работает, но MP3 работает

MP4 требует подходящий видео+аудио поток от `yt-dlp` и может обрабатываться дольше, потому что `ffmpeg` перекодирует фрагмент в H.264/AAC. Если выбранного качества нет, приложение просит `yt-dlp` подобрать ближайший совместимый поток. Если YouTube не отдаёт совместимый progressive-поток или сеть/VPN мешают загрузке, MP3 может продолжать работать отдельно.

### Thumbnail не скачивается

- Убедитесь, что metadata уже загружены и у ролика есть thumbnail.
- Выберите папку сохранения.
- Проверьте сеть/VPN: thumbnail скачивается по HTTP/HTTPS-ссылке из metadata.

### Export работает, но preview нет

Это возможно. Preview использует локальный streaming proxy поверх media URL, полученного через `yt-dlp`, но воспроизведение всё равно зависит от browser media engine внутри Electron. Export использует `ffmpeg` и может работать даже тогда, когда preview не воспроизводится в `<video>`.

В этом случае можно выбрать start/end вручную или через timeline после metadata, затем экспортировать MP3 или MP4.

### Экспорт работает в dev-режиме, но не работает в собранном приложении

Чаще всего собранное приложение запущено из среды, где не заданы переменные окружения или отличается `PATH`.

Минимальная проверка:

```powershell
$env:YT_DLP_PATH="C:\Tools\yt-dlp\yt-dlp.exe"
$env:FFMPEG_PATH="C:\Tools\ffmpeg\bin\ffmpeg.exe"
$env:FFPROBE_PATH="C:\Tools\ffmpeg\bin\ffprobe.exe"
.\release\win-unpacked\SiGame Media Cutter.exe
```

Если приложение собрано portable `.exe`, запускайте его из того же PowerShell, где заданы переменные окружения.

### Desktop-сборка просит подпись, иконку или падает на winCodeSign

В текущей MVP-конфигурации подпись и редактирование Windows executable отключены. Если вы включите их позже, могут понадобиться сертификат, иконка и права Windows на создание symlink. Для локального тестирования используйте текущие команды `npm run package` и `npm run dist` без code signing.

## Текущее состояние

MVP реализует один вертикальный сценарий: получить метаданные YouTube-видео, вручную указать границы фрагмента, экспортировать MP3/MP4 в выбранную папку и при необходимости скачать thumbnail.

Таймкоды проверяются на пустые значения, нечисловые значения, `start >= 0`, `end > start`, минимальную длину фрагмента и выход за пределы длительности, если длительность известна.

Есть preview-плеер, timeline без waveform, точный ввод времени до миллисекунд и проигрывание выбранного отрезка.

Пока нет waveform, очереди, сборки `.siq`/пакетов SiGame, резки локальных файлов и интеграций с другими источниками.
