## 2026-06-28: Первый vertical slice генерации `.siq`

### Что изменено

- Добавлен `ZipArchiveService` в main-process: простой ZIP writer без внешних зависимостей, достаточный для первого `.siq` slice.
- Добавлен `SiqPackageService`, который создаёт минимальный `.siq` пакет с `content.xml` и одним вложенным медиафайлом.
- Добавлен typed IPC/preload метод `createSiqPackage`.
- В правую панель UI добавлен компактный блок "Создать .siq" после успешного MP3/MP4-экспорта.
- UI создаёт один пакет с одним раундом, одной темой, одним вопросом, ценой, ответом и последним экспортированным MP3/MP4.
- README, `AGENTS.md` и `SIQ_INTEGRATION_NOTES.md` обновлены под текущее состояние.
- Renderer по-прежнему не пишет ZIP/XML и не читает медиафайлы напрямую; генерация `.siq` остаётся в main-process.
- Git-команды, меняющие состояние репозитория, не запускались.

### Как тестировать

```powershell
npm run typecheck
npm run build
npm run package
```

Ручная проверка:

1. Запустить приложение.
2. Получить metadata для разрешённого тестового YouTube-ролика.
3. Экспортировать короткий фрагмент в MP3 или MP4.
4. Заполнить блок "Создать .siq": название пакета, тему, текст вопроса, ответ и цену.
5. Создать `.siq` в выбранной папке.
6. Открыть `.siq` в SiQuester и проверить, что пакет открывается, вопрос виден, ответ и цена сохранены, медиа воспроизводится.

### Оставшиеся ограничения

- Совместимость с SiQuester пока нужно подтвердить вручную на созданном `.siq`.
- MP4/video-вопросы ещё нужно сверить с минимальным `.siq`, созданным самим SiQuester.
- Нет импорта существующих `.siq`.
- Нет полноценного редактора раундов, тем и вопросов.
- ZIP создаётся без сжатия; для MVP это допустимо, но позже можно заменить на библиотеку или добавить сжатие.

### Как запускать и тестировать

1. Установить Node.js 20 или новее.
2. Установить `yt-dlp`.
3. Установить `ffmpeg` и убедиться, что `ffprobe` тоже доступен.
4. Проверить инструменты в PowerShell:

   ```powershell
   yt-dlp --version
   ffmpeg -version
   ffprobe -version
   ```

5. Установить зависимости:

   ```powershell
   npm install
   ```

6. Запустить приложение:

   ```powershell
   npm run dev
   ```

7. Проверить MVP:
   - вставить ссылку YouTube на медиа, которое пользователь имеет право использовать;
   - получить метаданные;
   - указать отметки времени, например `0:15` и `0:45`;
   - выбрать папку сохранения;
   - экспортировать MP3.

### Известные ограничения

- В MVP поддерживается только YouTube.
- Временные отметки вводятся вручную.
- Имена файлов генерируются из названия медиа и временных отметок.

- Некоторые форматы YouTube или сетевые условия могут приводить к ошибкам получения потока, проверки длительности или экспорта.
- Нет интеграции с Yandex Music.
- Нет авторизации.

## 2026-06-25: Реальные пути winget-пакетов для внешних инструментов

### Что изменено

- `ToolPathResolver` теперь разворачивает существующие пути через `realpathSync.native`.
- `npm run check:tools` тоже показывает и проверяет реальные `.exe` внутри `%LOCALAPPDATA%\Microsoft\WinGet\Packages`, а не только ссылки из `%LOCALAPPDATA%\Microsoft\WinGet\Links`.
- Ошибки “Не найден yt-dlp/ffmpeg/ffprobe” теперь включают путь, который приложение пыталось запустить.

### Почему

Собранное Electron-приложение может запускаться в окружении, где winget-ссылка видна, но запуск через ссылку работает иначе, чем из PowerShell. Реальный target `.exe` надёжнее для `child_process.spawn`.

```powershell
npm run build
npm run package
```
