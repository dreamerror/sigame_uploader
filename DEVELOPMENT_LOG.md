# Журнал разработки

## 2026-06-24: Начальный MVP

### Что реализовано

- Создана структура Electron + Vue 3 + TypeScript + electron-vite.
- Добавлен типизированный preload API: renderer общается с main-процессом через `window.sigameApi`.
- Добавлены сервисы main-процесса:
  - `YtDlpService`
  - `FfmpegService`
  - `MediaProbeService`
  - `ExportService`
- Добавлены общие TypeScript-типы `MediaMetadata` и `MediaCutRequest`.
- Реализована загрузка метаданных через `yt-dlp`.
- Реализована ручная валидация временных отметок.
- Реализован экспорт MP3 через аудиопоток, полученный `yt-dlp`, и `ffmpeg`.
- Добавлен выбор папки сохранения через dialog API Electron.
- Добавлены состояния успеха и ошибки в UI.
- Добавлена обработка ошибок для отсутствующих инструментов, некорректной ссылки, некорректных отметок времени, ошибок метаданных и ошибок экспорта.

### Как запускать и тестировать

1. Установить Node.js 20 или новее.
2. Установить `yt-dlp`.
3. Установить `ffmpeg` и убедиться, что `ffprobe` тоже доступен.
4. Проверить инструменты в PowerShell:

   ```powershell
   yt-dlp --version
   ffmpeg -version
   ffprobe -version
   ```

5. Установить зависимости:

   ```powershell
   npm install
   ```

6. Запустить приложение:

   ```powershell
   npm run dev
   ```

7. Проверить MVP:
   - вставить ссылку YouTube на медиа, которое пользователь имеет право использовать;
   - получить метаданные;
   - указать отметки времени, например `0:15` и `0:45`;
   - выбрать папку сохранения;
   - экспортировать MP3.

### Известные ограничения

- В MVP поддерживается только YouTube.
- Временные отметки вводятся вручную.
- Имена файлов генерируются из названия медиа и временных отметок.
- Нет waveform, предпрослушивания, очереди, сборщика пакетов SiGame и генерации XML/структуры `.siq`.
- Приложение ожидает локально установленные `yt-dlp`, `ffmpeg` и `ffprobe`.
- Некоторые форматы YouTube или сетевые условия могут приводить к ошибкам получения потока, проверки длительности или экспорта.
- Нет интеграции с Yandex Music.
- Нет авторизации.

### Рекомендуемые следующие шаги

- Добавить небольшой экран настроек для явного указания путей к инструментам.
- Добавить предпрослушивание и проверку выбранного фрагмента.
- Добавить более аккуратную обработку совпадений имён выходных файлов.
- Добавить тесты для парсинга временных отметок и маппинга ошибок сервисов.
- Добавить скрипты упаковки для Windows после стабилизации MVP-сценария.

Собранное Electron-приложение может запускаться в окружении, где winget-ссылка видна, но запуск через ссылку работает иначе, чем из PowerShell. Реальный target `.exe` надёжнее для `child_process.spawn`.


```powershell
npm run check:tools
npm run typecheck
npm run build
npm run package
```

Ручная проверка:

1. Запустить приложение.

- Thumbnail скачивается только по URL из metadata; отдельный выбор качества thumbnail пока не добавлен.
- Git намеренно не трогался.
